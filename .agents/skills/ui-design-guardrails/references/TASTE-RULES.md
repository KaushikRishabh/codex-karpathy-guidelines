# Taste rules

These are positive design rules to bias Codex toward higher-quality product UI decisions.

## Good taste in this repo means
- strong hierarchy
- calm surfaces
- confident whitespace
- one obvious next action
- consistency across screens
- UI that feels intentional, not ornamental

## Preferred choices
- simpler grid or column layouts
- fewer containers
- restrained use of borders, shadows, and color
- concise labels and concise helper text
- visual emphasis used sparingly and deliberately
- clear primary buttons and quieter secondary actions

## When in doubt
- remove one decorative layer
- simplify one container
- shorten one label
- make the main CTA easier to find
- reduce visual noise before adding new elements
