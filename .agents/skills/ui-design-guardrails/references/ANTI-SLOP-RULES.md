# Anti-slop rules

Use this file as a negative filter when making UI changes.

## Avoid by default
- large blurred gradient backgrounds
- floating translucent glass cards
- giant rounded corners on every component
- heavy stacked shadows everywhere
- center-aligned marketing layouts for product flows that should be task-focused
- decorative icons on every card with no functional value
- overly padded layouts that hide the real action below the fold
- tiny low-contrast helper text
- 3+ accent colors in one view
- long vague CTA text
- split attention across multiple equal-weight CTAs

## Bad signs
- The screen looks "fancy" but the next action is unclear
- The user has to search for the main button
- The design uses style to compensate for weak hierarchy
- Every surface tries to be a hero element

## Preferred replacements
- stronger spacing hierarchy
- one clear primary CTA
- restrained color use
- subtle radius and subtle shadow
- simpler cards or no card at all
- better labels and tighter copy
