# Design review checklist

Use this checklist when editing UI.

## Product clarity
- Can a new user understand the screen quickly?
- Is the main action obvious?
- Is there one primary CTA?

## Hierarchy
- Is the heading clear?
- Is supporting text concise?
- Is the most important information visually dominant?

## States
- Is there a loading state if the user must wait?
- Is there an empty state if there may be no data?
- Is there an error state if something fails?
- Is there a success state when an action completes?

## Layout
- Is spacing consistent with DESIGN.md?
- Is the page free of unnecessary nesting?
- Does the layout remain usable on mobile?

## Components
- Are buttons clearly prioritized?
- Are forms easy to scan?
- Are card/list/table layouts easy to parse?

## Quality
- Are visual changes limited to the task scope?
- Does the screen feel cleaner rather than just more decorative?
- Would a human product designer call this calmer and clearer?
