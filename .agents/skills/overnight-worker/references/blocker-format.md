# Blocker format

Use this structure when a task cannot be completed safely.

## Task
- task id / name

## Goal
- one or two lines restating the goal

## Blocking issue
- what prevented completion

## What was tried
- concise list of attempted fixes or checks

## Current state
- whether the repo is clean
- whether there is a useful patch saved by the runner

## Recommended next step
- what the user should decide, provide, or fix before retrying

## Risk note
- any important warning about scope, missing credentials, or unstable areas
