# Summary format

Use this structure for a completed overnight task summary.

## Task
- task id / name

## Goal
- one or two lines restating the goal

## What changed
- concise bullet list of the actual code changes

## Files touched
- list of changed files or key files

## Verification
- commands run
- focused manual checks performed
- what passed
- what was not verified

## Notes
- any important implementation tradeoffs
- any follow-up risk that the morning review should inspect

## Commit
- commit hash
